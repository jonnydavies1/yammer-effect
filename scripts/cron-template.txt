# yammer-scan cron entry (Linux)
#
# IMPORTANT: this template is provided as a scheduling reference, but
# scripts/yammer-scan-template.sh as shipped will NOT run on Linux without
# porting. It uses macOS-specific calls:
#   - `security` (macOS Keychain) for token storage
#   - `stat -f %m` (BSD) for folder mtime
#   - `date -r <epoch>` and `date '+...'` flags that diverge from GNU coreutils
#   - log/cache paths under `~/Library/...`
# Replace those before relying on this cron entry. v1 is dogfooded on macOS
# only; Linux porting is a known follow-up.
#
# Install (after porting):
#   crontab -l > /tmp/cron.tmp 2>/dev/null
#   cat cron-template.txt >> /tmp/cron.tmp
#   crontab /tmp/cron.tmp
#
# Remove: crontab -e and delete the relevant lines
#
# The 12h debounce in the script means multiple triggers per day are safe
# (only one will actually scan, the rest are no-ops).
#
# Tokens: ensure the script uses an env file or libsecret/keyring rather
# than cron-time interactive auth. cron has minimal env, so $PATH and
# any tool locations need to be explicit.

PATH=/usr/local/bin:/usr/bin:/bin:/snap/bin

# 06:00, 09:00, 13:00 daily
0 6,9,13 * * * /home/USERNAME/bin/yammer-scan.sh >> /home/USERNAME/.local/state/yammer/scan.log 2>&1
